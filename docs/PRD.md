# PRD.md — Plant Watering Tracker

**Версия:** 0.2  
**Дата:** 2026-06-08  
**Статус:** Утверждено

---

## 1. Цель

Простое PWA-приложение для отслеживания полива домашних растений.  
Пользователь видит список растений и историю полива за 7 дней,  
отмечает дни полива, приложение подсказывает когда пора поливать снова.

---

## 2. Платформа

- **PWA** — устанавливается на iOS и Android через браузер
- **Хостинг:** GitHub Pages (статика, без бэкенда)
- **Офлайн:** работает без интернета через Service Worker
- **Данные:** localStorage + ручной экспорт/импорт JSON

---

## 3. Главный экран

### Шапка
- Название приложения
- Кнопка меню (⋮) → экспорт / импорт данных

### Список растений
Каждая строка:
```
[Название растения]   [○][○][○][○][○][○][●]
                      -6  -5  -4  -3  -2  -1  сег
```

- Порядок строк — в порядке создания (пользовательский)
- Нажатие на **название** → модалка редактирования
- Нажатие на **кружочек** → toggle (полито ↔ не полито)

### FAB
- Кнопка **[+]** в правом нижнем углу
- Открывает модалку создания нового растения

---

## 4. Состояния кружочков

| Состояние | Фон кружочка | Цвет даты | Условие |
|---|---|---|---|
| Полито | акцентный цвет (зелёный) | белая | `wateredDates` содержит эту дату |
| Полив не нужен | серый | серая | `day < nextWatering` |
| Пора поливать | яркий акцент | яркая | `day == nextWatering` |
| Просрочено | красный | красная | `day > nextWatering` и не полито |

> Приоритет: **полито** перекрывает все остальные состояния.

---

## 5. Логика полива

```
frequencyDays = frequencyUnit == "weeks"
  ? frequencyValue * 7
  : frequencyValue

lastWatered = max(wateredDates)  // последняя дата полива
nextWatering = lastWatered + frequencyDays

// Для каждого кружочка (дата D):
если D в wateredDates        → ПОЛИТО
иначе если D < nextWatering  → СЕРЫЙ
иначе если D == nextWatering → ЯРКИЙ
иначе (D > nextWatering)     → КРАСНЫЙ
```

Если `wateredDates` пуст — все кружочки яркие (нужен полив).

## 6. Модалка создания / редактирования

### Открытие
- FAB (+) → режим **создания**
- Нажатие на название растения → режим **редактирования**

### Поля
| Поле | Тип | Обязательное |
|---|---|---|
| Название растения | text input | да |
| Периодичность — число | number input (1–365) | да |
| Периодичность — единица | select: дни / недели | да |

### Кнопки
- **Сохранить** — создать / обновить запись
- **Отмена** — закрыть без изменений
- **Удалить** — только в режиме редактирования, с диалогом подтверждения

## 7. Экспорт / импорт данных

Доступ: меню (⋮) в шапке.

### Экспорт
- Скачать файл `watering-backup-YYYY-MM-DD.json`
- Формат: JSON-массив Plant[]

### Импорт
- Выбрать `.json` файл
- Показать предупреждение: «Текущие данные будут заменены»
- Подтверждение → загрузить данные

---

## 8. Структура данных

```typescript
type FrequencyUnit = "days" | "weeks";

interface Plant {
  id: string;           // UUID
  name: string;         // название растения
  frequencyValue: number; // число
  frequencyUnit: FrequencyUnit;
  wateredDates: string[]; // ISO dates: "YYYY-MM-DD"
  createdAt: string;    // ISO date
}
```

localStorage ключ: `"watering_plants"`  
Значение: `JSON.stringify(Plant[])`
